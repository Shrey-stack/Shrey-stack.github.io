Linux File System

Linux file system has a hierarchal file structure as it contains a root directory and its subdirectories. The Linux directory structure is like a tree. The base of the Linux file system hierarchy begins at the root. Directories branch off the root, but everything starts at root. The directory separator in Linux is the forward slash (/). Often the root of the file system is referred to as “slash” since the full path to it is /. If you hear someone say “look in slash” or “that file is in slash,” they are referring to the root directory.

Let's take a look into the file system in brief


/root
The root filesystem is the top-level directory of the filesystem. It must contain all of the files required to boot the Linux system before other filesystems are mounted. It must include all of the required executables and libraries required to boot the remaining filesystems. After the system is booted, all other filesystems are mounted on standard, well-defined mount points as subdirectories of the root filesystem.

/bin
The /bin directory contains user executable files. The core command of Linux such as mv, ls can be found here.

/boot
The /boot directory Contains the static bootloader and kernel executable and configuration files required to boot a Linux computer.

/dev
This directory contains the device files for every hardware device attached to the system. These are not device drivers, rather they are files that represent each device on the computer and facilitate access to those devices.

/etc
This directory contains the local system configuration files for the host computer. It contains configurations for the installed packages.

/home
This directory contains user home directories. It allows each user to separate their data from the other users on the system.

/lib
This directory contains shared library files that are required to boot the system.

/media
This directory is a place to mount external removable media devices such as USB thumb drives that may be connected to the host.

/lib
This directory contains shared library files that are required to boot the system.

/opt
This directory contains Optional files such as vendor supplied application programs.

/proc
This directory contains all the information of process that is running. It contains virtual file system that kernel uses to communicate with the system and other programme in the machine.

/sbin
This directory contains system binary files. These are very critical files which the machine needs for running. Please do not modify it unless necessary.

/tmp
This is a temporary directory.It is used by the operating system and many programs to store temporary files. The files stored here may be deleted at any time without prior notice.

/usr
This directory contains shareable, read-only files, including executable binaries and libraries, man files, and other types of documentation.

I hope by now you have a good idea about what the Linux file system is.

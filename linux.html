Linux

Few reasons why linux is better
* In Linux user has access to the source code of kernel and alter the code according to his need. It has its own advantages like bugs in OS will fix at a rapid pace and disadvantages like developers may take advantage of any weakness in OS if they found.
* Linux has various distributions which are highly customizable based on user needs.
* In Linux, command line is a very useful tool for administration and daily tasks.
* In Linux, users have full control of updates, we can install whenever we needed and it will take less time without any reboot.
* Linux is more secure and hackers or developers of viruses will find difficult to break through Linux.

Useful commands in Linux:


I have listed some basic Linux commands below:
To print working directory
# pwd
To list things in directory
# ls
To change directory
# cd
To go to root
# / or cd / or ls /
to list hidden files and non hidden files
# ls -a
to create new file
# touch newfilename.txt
To create new directory (folder)
# mkdir newfoldername
To move file
# mv filename(source) target(where you want the file to be)
To delete file
# rm filename
To delete directory
# rmdir foldername
To delete directory with contents
# rm -r foldername
To display the user manual of any command
# man command (ex man vim)
to view long listing of the files including permisiion and sizes in human readable format
# ls -alh
To go to nano file editor
# nano filename
To execute command after the previous command is successful
# && (ex prog1 && prog2)

Other useful commands


uname -a
This command is used to check kernel version of current system. It shows host name, OS, Kernel, Kernel version, Release

ifconfig
This command shows the IPV4 address, IPV6 address, MAC address and Sub-net masks.

df -ah
This command checks for free disk space. Here, SDA 1 is the root portion.

systemctl status service_name
This command checks how a service is managed in the system. Below, I have used udev service as an example.

du -sh -file/folder_name
This command checks the size of a directory or a file's content on a disk.
netstat
This command checks for open ports, both tcp and udp. However, it shows a very long list.
It is highly suggested to use netstat_tulpn. This command shows Local address, Foreign Address, State and PID in a shortlisted format.

man command
As mentioned above, this command shows the manual/ guidelines of anything you want to know. Below, I am using man for grep.
